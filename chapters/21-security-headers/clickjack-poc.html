Clickjacking proof of concept

The translucent box below is an iframe to your application. The yellow "Win!" overlay sits on top. The user thinks they're clicking the prize button — they're actually clicking whatever is at that position in the iframe (e.g. a "Confirm transfer" button on a banking page).

With X-Frame-Options: DENY or Content-Security-Policy: frame-ancestors 'none' in place, the iframe refuses to load and this attack fails entirely.

Open the browser console after loading this file. If the iframe loaded, you'll see no errors. If it was blocked, you'll see one of:

Refused to display 'http://localhost:8080/' in a frame because it set 'X-Frame-Options' to 'deny'.
Refused to frame 'http://localhost:8080/' because an ancestor violates the following Content Security Policy directive: "frame-ancestors 'none'".
